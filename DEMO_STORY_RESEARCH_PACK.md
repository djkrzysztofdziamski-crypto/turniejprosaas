# Demo Story — Research Pack v1.0

**Wersja:** 1.0  
**Status:** Gotowy do użycia w terenie  
**Data:** 2026-07-12  
**Hipoteza centralna:** Organizator po przejściu Demo Story mówi: *„Chcę taki turniej u siebie.”*  
**Pytanie biznesowe:** Czy Demo Story **zwiększa chęć zakupu** TurniejPro SaaS?

**Powiązane dokumenty:** `DEMO_STORY_VALIDATION_PLAN.md` · `COPY_DECK_DEMO_STORY.md` · `PRD_DEMO_STORY.md`

**Zespół sesji:** 1 moderator + 1 obserwator (opcjonalnie nagrywający).  
**Czas sesji:** 15–20 minut.  
**Liczba uczestników:** 3–5 organizatorów turniejów halowych (~16 drużyn).

---

# Część 1 — Moderator Guide

## 1.1 Rola moderatora

| Robisz | Nie robisz |
|--------|------------|
| Wprowadzasz kontekst i zasady think-aloud | Nie tłumaczysz, jak działa produkt |
| Zadajesz pytania w punktach E5 i E7 | Nie podpowiadasz, gdzie klikać |
| Notujesz cytaty dosłowne | Nie sprzedajesz ani nie bronisz produktu |
| Interweniujesz tylko przy blokadzie >2 min | Nie przerywasz emocji na E5 |

**Entry point dla uczestnika:** ekran logowania → przycisk **„ZOBACZ FINAŁ TURNIEJU”**.

---

## 1.2 Przed sesją (checklist 5 min)

- [ ] Demo działa na urządzeniu uczestnika (telefon preferowany)
- [ ] Obserwator ma wydrukowany **Observer Sheet** (Część 2)
- [ ] Stoper / zegarek do pomiaru czasu start → E5 → E7
- [ ] Zgoda na nagranie (jeśli dotyczy)
- [ ] Uczestnik **nie** widział wcześniej Demo Story ani DEMO-2026

---

## 1.3 Scenariusz rozmowy (15–20 min)

### A. Rozpoczęcie (2 min)

**Powiedz:**

> „Dziękuję, że przyszedłeś. Dzisiaj pokażę Ci krótką historię turnieju — około 2–3 minut interakcji.  
> To nie jest egzamin ani szkolenie. Chcę usłyszeć, co myślisz na głos, krok po kroku.  
> Nie ma złych odpowiedzi. Jeśli coś jest niejasne — powiedz to.  
> Możesz używać telefonu jak zwykle. Zacznij, kiedy będziesz gotowy.”

**Zapisz:** zgodę na nagranie T/N · urządzenie · godzina startu.

**Nie mów:** co to TurniejPro, ile ekranów, że „to demo SaaS”.

---

### B. Kontekst organizatora (2 min)

**Powiedz:**

> „Zanim zaczniemy — dwie krótkie rzeczy o Tobie, żebym zrozumiał kontekst.”

**Pytania (zapisz odpowiedzi):**

1. *„Ile drużyn zwykle masz na turnieju?”*
2. *„Kto u Ciebie wpisuje wyniki w dniu turnieju?”*
3. *„Co jest najbardziej męczące pod koniec dnia turnieju?”*

**Cel:** Ustalić, czy rozmawiamy z ICP (12–20 drużyn, ból końcówki dnia / rodzice).

**Przejście:**

> „Super. Teraz proszę — wejdź na stronę, którą Ci otworzyłem, i przejdź historię po swojemu. Myśl na głos.”

---

### C. Przejście przez Demo Story (5–10 min)

**Moderator milczy.** Obserwator wypełnia arkusz.

**Interwencja dozwolona tylko gdy:**

- Uczestnik stoi >2 min bez akcji → *„Co teraz widzisz? Co byś zrobił?”*
- Pyta o hasło / klucz → *„Możesz wybrać opcję bez klucza — demo.”*
- Pyta „co mam wpisać?” na finałe → *„Dowolny wynik, który uważasz za sensowny.”*

**Notuj:**

| Moment | Czas (od startu) |
|--------|------------------|
| Klik start demo (E0→E1) | |
| Wejście E2 (kibic) | |
| Wejście E5 (podium) | |
| Zapis finału (E4) | |
| Wejście E6 (konwersja) | |
| Wejście E7 (archiwum) | |

**Checkpoint E5:** Gdy uczestnik **po raz pierwszy** zobaczy podium — **zatrzymaj** na 2–3 min (sekcja D), nawet jeśli chce iść dalej.

> *„Zatrzymajmy się na chwilę tutaj — kilka pytań o to, co właśnie widzisz.”*

Po sekcji D pozwól dokończyć E6→E7 samodzielnie, jeśli jeszcze nie przeszedł.

---

### D. Pytania po E5 — Peak Moment (3 min)

**Kontekst:** Uczestnik widzi podium po wpisaniu finału. Emocja jest „świeża”.

| # | Pytanie moderatora | Notatka |
|---|-------------------|---------|
| D1 | *„W jednym zdaniu: co właśnie się stało w turnieju?”* | |
| D2 | *„Na skali 1–5: jak bardzo wierzysz, że po prawdziwym finałe nie musiałbyś liczyć podium na kartce?”* 1= wcale · 5= całkowicie | /5 |
| D3 | *„Co powiedziałbyś rodzicom — skąd mają wyniki na telefonie?”* | |
| D4 | *„Co według Ciebie sprzedaje TurniejPro — na podstawie tego, co widziałeś?”* | **Kluczowe pod H1** |
| D5 | *„Gdyby to był Twój turniej — czy ten moment po finałe byłby dla Ciebie ważny?”* Dlaczego? | |

**Sonda opcjonalna (jeśli pozytywnie):**

> *„Czy mógłbyś powiedzieć: «Chcę taki turniej u siebie» — tak czy nie? Co by musiało być inaczej?”*

**Powiedz przed wznowieniem:**

> „Dzięki. Dokończ proszę resztę historii — jeszcze chwilę.”

---

### E. Pytania po E7 — Konwersja i decyzja (5 min)

**Warunek:** Uczestnik dotarł do archiwum LUB zakończył flow na E6.

| # | Pytanie moderatora | Notatka |
|---|-------------------|---------|
| E1 | *„Na skali 1–5: czy chciałbyś, żeby Twój następny turniej wyglądał tak samo?”* | /5 · **H6** |
| E2 | *„Co było dla Ciebie najważniejsze w tej historii?”* | |
| E3 | *„Co było niejasne, zbędne lub irytujące?”* | |
| E4 | *„Ekran «Twój turniej może wyglądać tak samo» — był we właściwym momencie?”* Tak / Nie / Nie wiem | |
| E5 | *„Który przycisk byś kliknął: aktywacja licencji, zamówienie klucza, oferta — czy żaden?”* | |
| E6 | *„Co musiałoby się stać, żebyś kupił lub aktywował to przed swoim turniejem?”* | |
| E7 | *„Gdybyś polecał to koledze-organizatorowi — co powiesz jednym zdaniem?”* | |

**Pytanie zamykające hipotezę:**

> *„Czy po tej historii **chcesz taki turniej u siebie**? Proszę o szczerą odpowiedź — tak, może, nie.”*

| Odpowiedź | Zapisz |
|-----------|--------|
| Tak — bez zastrzeżeń | |
| Może — z warunkami (jakie?) | |
| Nie — dlaczego? | |

---

### F. Zakończenie (1 min)

**Powiedz:**

> „Dziękuję — to było bardzo pomocne. Twoje uwagi wpływają bezpośrednio na to, jak opowiadamy historię produktu.  
> Nie musisz teraz nic kupować. Jeśli chcesz przetestować na swoim turnieju — daj znać, odezwiemy się.”

**Po sesji (moderator + obserwator, 5 min):**

- Uzupełnij **Participant Scorecard** (Część 3)
- Przenieś dane do **Results Summary** (Część 4)
- Zaznacz sygnały ostrzegawcze na Observer Sheet
