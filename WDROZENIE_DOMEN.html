Dwa osobne site'y Netlify

Musisz mieć dwa deploye — landing i aplikacja nie mogą dzielić jednego index.html na apex.

  1. Site A — Landing (turniejomat.pl)

    Otwórz app.netlify.com → Add new site → Deploy manually.

    Przeciągnij cały folder:

    turniejprosaas\landing\

    (zawiera: index.html, logo.png, icon.png, netlify.toml)

    Albo podłącz repo GitHub z ustawieniem:

    • Base directory: landing
    • Publish directory: landing (lub . jeśli base = landing)

    W Domain management dodaj:

    • turniejomat.pl
    • www.turniejomat.pl

    Skonfiguruj DNS wg instrukcji Netlify (rekord A / CNAME).

    Jeśli masz już site stronaturniejomat.netlify.app — możesz zaktualizować ten site zawartością folderu landing/ zamiast tworzyć nowy.

  2. Site B — Aplikacja (app / demo / admin)

    To Twój obecny site turniejprosaas z GitHub.

    W Domain management upewnij się, że są:

    • app.turniejomat.pl
    • demo.turniejomat.pl
    • admin.turniejomat.pl
    Ważne: usuń turniejomat.pl z tego site'u aplikacji (jeśli tam jest) — apex ma wskazywać tylko na Site A (landing).

    Deploy aplikacji: na razie lokalne zmiany nie są na GitHub. Gdy będziesz gotowy — poprosisz o commit+push, wtedy Netlify zbuduje nową wersję.

  3. Okres przejściowy (zanim apex przejdzie na landing)

    Jeśli turniejomat.pl nadal wskazuje na stary deploy aplikacji, kod tymczasowo przekierowuje gości na:

    https://stronaturniejomat.netlify.app/

    Stare linki z kluczem ?id=TP-... trafią na app.turniejomat.pl.

    Po podpięciu landingu pod apex — to przekierowanie nie będzie już potrzebne (aplikacja nie dostaje ruchu z apex).

  4. Test po wdrożeniu

    • turniejomat.pl — landing, cennik 79/149, przycisk demo
    • demo.turniejomat.pl — od razu E0 „Zobacz finał”
    • app.turniejomat.pl — bramka klucza, link demo na zewnątrz
    • admin.turniejomat.pl — panel admina
    • Przycisk „Zamów pakiet” — otwiera maila do admin@turniejomat.pl
    • Demo E6 → „Aktywuj licencję” → app.turniejomat.pl
